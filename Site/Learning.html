CSS

It is all about the presentation of the content, so it is understandable.

Anatomy of a CSS Rule

It is in the following way:

p{
color: blue;
width: 200px;
}

There is usually a stylesheet, where you add all these rules. It can be directly applied in the head of the html. Check the code There are some presets on the styles, so with CSS you rewrite them.

Element, Class and ID Selectors

  • Element Selector. You write the tag name. It will apply the style to all the corresponding tags. p{color:blue;}
  • Class Selector. Create a class to apply to certain stuff. .blue{color:blue;} Have to indicate in the opening tag the corresponding class without "."
  • id Selector. Choose the id name of something and space the value. #name{color:blue;}

We can group selectors, separating by a comma

Combining Selectors

Technique to combine elements. For example:

  • p.big- It will affect all paragraphs with this class
  • article > p- Means every paragraph inside an article, as long as it is a direct child.
  • article p- any p inside article, doesn´t matter what level.
  • To specify two conditions that need to be met, such as two classes do: .class1.class2

It applies to any selector, also classes. Applying various classes to an element is done by: class:class1 class2;

Pseudo-Class Selector

Address elements that can´t be targeted with the normal selectors. An example is an element that changes when the mouse hovers on top of it. To specify it:

selector:pseudo-class{...}

Some cases of pseudo-class selctors are:

  • :link
  • :visited
  • :hover
  • :active
  • :nth-child

Conflict Resolution

Cascading is an algorithm used in CSS. It tells which rule wins over an element. They are the following:

  • Origin.
    • The last declaration wins. From top to bottom.
    • If no conflict, declarations merge.
  • Inheritance. If you declare a property to one element, the children will also have that property.
  • Specificity. Most specific selector combiantion wins. The order on specificity is:
    • style="..."
    • ID
    • Class, pseudo-class, attribute
    • # of Elements

    • Check in depth when not having the desired color. If you want some property no matter what, add !important next to it-- color:green !important;

    Avoid using important unless no other choice, as mantainance gets more difficult.
